1.1 类型推导与初始化

auto 类型推导

auto 是C++11引入的关键字，让编译器根据初始化表达式自动推导变量类型，减少冗长的类型书写。

基本用法：

1
2
3
4
5
6
7
8
9

auto i = 42;           // int
auto d = 3.14;         // double
auto s = std::string("hello"); // std::string

// 迭代器场景特别有用
std::vector<int> v = {1, 2, 3};
for (auto it = v.begin(); it != v.end(); ++it) {
    std::cout << *it << "\n";
}

推导规则：

  • auto 推导时会忽略顶层const和引用： 
    1
2
3

    const int x = 10;
auto a = x;   // a 是 int，不是 const int（顶层const被丢弃）
auto& b = x;  // b 是 const int&（引用保留const）
  • 数组和函数会退化为指针： 
    1
2

    int arr[3] = {1, 2, 3};
auto p = arr;  // p 是 int*，不是 int[3]

限制（不能使用auto的场景）：

  • 函数参数不能是auto（C++20之前）
  • 类的非静态成员变量不能用auto直接声明
  • 不能用auto声明但不初始化

decltype 说明符

decltype 用于查询表达式的类型，不会实际求值表达式，只获取其类型信息。

1.3 右值引用与移动语义

左值与右值的概念

  • 左值（lvalue）：有名字、有稳定内存地址的表达式，可以取地址，如变量名、解引用、数组元素
  • 右值（rvalue）：临时对象、字面量，通常没有持久内存地址，不能取地址
    • 纯右值（prvalue）：字面量（42、3.14）、函数返回的非引用类型
    • 将亡值（xvalue）：即将被销毁的对象，如 std::move(x) 的结果、返回右值引用的函数结果
1
2
3
4

int x = 10;       // x 是左值，10 是纯右值
int& lref = x;    // 左值引用，只能绑定左值
int&& rref = 10;  // 右值引用，只能绑定右值
// int&& r2 = x;  // 错误！右值引用不能绑定左值

右值引用与万能引用

右值引用 T&&（T是具体类型）只绑定右值，其核心用途是支持移动语义和完美转发。

万能引用（Universal Reference）：当 T&& 中的 T 是需要类型推导的模板参数时，它既能绑定左值也能绑定右值：

规律：只要有一个左值引用，结果就是左值引用。

class Buffer {
public:
    // 普通构造
    Buffer(size_t size) : size_(size), data_(new char[size]) {}

    // 拷贝构造（深拷贝，O(n)）
    Buffer(const Buffer& other) : size_(other.size_), data_(new char[other.size_]) {
        std::copy(other.data_, other.data_ + size_, data_);
    }

    // 移动构造（O(1)，只转移指针）
    Buffer(Buffer&& other) noexcept
        : size_(other.size_), data_(other.data_) {
        other.size_ = 0;
        other.data_ = nullptr;  // 让原对象失去所有权
    }

    // 移动赋值运算符
    Buffer& operator=(Buffer&& other) noexcept {
        if (this != &other) {
            delete[] data_;       // 释放自己的旧资源
            size_ = other.size_;
            data_ = other.data_;
            other.size_ = 0;
            other.data_ = nullptr;
        }
        return *this;
    }

    ~Buffer() { delete[] data_; }

private:
    size_t size_;
    char* data_;
};

为什么移动函数要加 noexcept？

  • 标准库容器（如 std::vector）在扩容时，如果移动构造可能抛异常，会退回使用拷贝构造（为保证强异常安全）。标记 noexcept 才能触发真正的移动优化。

  • join()：主线程阻塞等待子线程结束，之后子线程资源被回收

  • 注意：std::thread 对象析构前必须调用 join() 或 detach()，否则程序会调用 std::terminate()

    for (int i = 0; i < 10; ++i) {

            std::lock_guard<std::mutex> lock(mtx);
            dataQueue.push(i);

// 消费者
void consumer() {
    while (true) {
        std::unique_lock<std::mutex> lock(mtx);
        // 等待条件：队列非空
        // wait 内部：解锁 → 阻塞等待 → 被唤醒 → 重新加锁 → 检查条件
        cv.wait(lock, []{ return !dataQueue.empty(); });

        int val = dataQueue.front();
        dataQueue.pop();
        // 使用 val...
    }
}

class ThreadPool {
public:
    explicit ThreadPool(size_t threads) {
        for (size_t i = 0; i < threads; ++i) {
            workers_.emplace_back([this] {
                while (true) {
                    std::function<void()> task;
                    {
                        std::unique_lock<std::mutex> lock(queueMutex_);
                        condition_.wait(lock, [this] {
                            return stop_ || !tasks_.empty();
                        });
                        if (stop_ && tasks_.empty()) return;
                        task = std::move(tasks_.front());
                        tasks_.pop();
                    }
                    task();  // 执行任务
                }
            });
        }
    }

    template<typename F, typename... Args>
    auto submit(F&& f, Args&&... args) -> std::future<decltype(f(args...))> {
        auto task = std::make_shared<std::packaged_task<decltype(f(args...))()>>(
            std::bind(std::forward<F>(f), std::forward<Args>(args)...)
        );
        std::future<decltype(f(args...))> res = task->get_future();
        {
            std::unique_lock<std::mutex> lock(queueMutex_);
            tasks_.emplace([task] { (*task)(); });
        }
        condition_.notify_one();
        return res;
    }

    ~ThreadPool() {
        { std::unique_lock<std::mutex> lock(queueMutex_); stop_ = true; }
        condition_.notify_all();
        for (auto& w : workers_) w.join();
    }

private:
    std::vector<std::thread> workers_;
    std::queue<std::function<void()>> tasks_;
    std::mutex queueMutex_;
    std::condition_variable condition_;
    bool stop_ = false;
};

class LRUCache {
    int capacity_;
    std::list<std::pair<int,int>> lst_;  // 双向链表，front是最近使用
    std::unordered_map<int, std::list<std::pair<int,int>>::iterator> map_;

public:
    LRUCache(int capacity) : capacity_(capacity) {}

    int get(int key) {
        if (!map_.count(key)) return -1;
        // 移到链表头部（最近使用）
        lst_.splice(lst_.begin(), lst_, map_[key]);
        return map_[key]->second;
    }

    void put(int key, int value) {
        if (map_.count(key)) {
            map_[key]->second = value;
            lst_.splice(lst_.begin(), lst_, map_[key]);
        } else {
            if ((int)lst_.size() == capacity_) {
                // 淘汰链表尾部（最久未使用）
                map_.erase(lst_.back().first);
                lst_.pop_back();
            }
            lst_.emplace_front(key, value);
            map_[key] = lst_.begin();
        }
    }
};

void quickSort(std::vector<int>& arr, int left, int right) {
    if (left >= right) return;
    
    // 三数取中优化，减少最坏情况
    int mid = left + (right - left) / 2;
    int pivot = arr[mid];
    
    int i = left, j = right;
    while (i <= j) {
        while (arr[i] < pivot) ++i;
        while (arr[j] > pivot) --j;
        if (i <= j) {
            std::swap(arr[i], arr[j]);
            ++i; --j;
        }
    }
    quickSort(arr, left, j);
    quickSort(arr, i, right);
}
// 平均 O(n log n)，最坏 O(n²)，不稳定，原地排序

// 构建 next 数组（失配函数）
// next[i] 表示 pattern[0..i] 中，最长公共前后缀的长度
std::vector<int> buildNext(const std::string& pattern) {
    int n = pattern.size();
    std::vector<int> next(n, 0);
    int j = 0;
    for (int i = 1; i < n; ++i) {
        while (j > 0 && pattern[i] != pattern[j]) j = next[j-1];
        if (pattern[i] == pattern[j]) ++j;
        next[i] = j;
    }
    return next;
}

// KMP 匹配
int kmpSearch(const std::string& text, const std::string& pattern) {
    auto next = buildNext(pattern);
    int j = 0;
    for (int i = 0; i < (int)text.size(); ++i) {
        while (j > 0 && text[i] != pattern[j]) j = next[j-1];
        if (text[i] == pattern[j]) ++j;
        if (j == (int)pattern.size()) {
            return i - j + 1;  // 匹配成功，返回起始位置
        }
    }
    return -1;
}

        // 从右往左遍历（保证每件物品只用一次）
        for (int j = W; j >= weight[i]; --j) {
            dp[j] = std::max(dp[j], dp[j - weight[i]] + value[i]);
        }
    }
    return dp[W];
}

3.4 算法复杂度分析

常见时间复杂度排序：

$$O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(2^n) < O(n!)$$

主定理（递归复杂度）：

$$T(n) = aT(n/b) + f(n)$$
  • $a$：子问题数量
  • $b$：规模缩小比例
  • $f(n)$：非递归部分的复杂度

快速排序：$T(n) = 2T(n/2) + O(n)$，解为 $O(n \log n)$

4.2 Redis

数据结构底层实现

SDS（简单动态字符串）：Redis字符串的底层实现，在C字符串基础上增加了 len（实际长度）和 alloc（分配空间）字段，避免每次 strlen 遍历，支持二进制安全（可存储 \0）。

跳表（Sorted Set）：

  • 分数不同：直接按分数排序
  • 分数相同：按字典序比较键名

压缩列表（ziplist）：小数量时使用紧凑的连续内存存储，节省空间但增删效率低。数量超过阈值后转换为更适合的结构。

持久化机制

RDB（Redis Database Backup）：

  • 将内存快照序列化到磁盘（.rdb 文件）
  • 使用 fork() + 写时复制（COW），父进程继续服务，子进程做快照
  • 优点：文件紧凑，恢复快；缺点：两次快照之间的数据可能丢失

AOF（Append Only File）：

  • 记录每条写命令，追加到 .aof 文件
  • fsync 策略：always（每次写同步，最安全）、everysec（每秒同步）、no（依赖OS）
  • 优点：数据更完整；缺点：文件大，恢复慢

缓存三大问题

问题 原因 解决方案
缓存穿透 查询不存在的数据，每次都穿透到数据库 缓存空值；布隆过滤器过滤非法key
缓存击穿 热点key过期，大量并发请求同时打到数据库 互斥锁（只允许一个线程重建缓存）；热点key不设过期时间
缓存雪崩 大量key同时过期，或Redis宕机，流量全打到数据库 过期时间加随机扰动；集群高可用；限流降级

4.3 MySQL

MVCC（多版本并发控制）

InnoDB通过 MVCC 实现高并发读写无锁冲突：

  • 每一行数据隐含两个字段：事务ID（trx_id） 和 回滚指针（roll_pointer）
  • 旧版本数据存储在 undo log 中，通过回滚指针链接（版本链）
  • 读操作根据**视图（Read View）**判断哪个版本可见：
    • 快照读（普通SELECT）：读取版本链中符合可见性规则的版本
    • 当前读（SELECT FOR UPDATE、INSERT/UPDATE/DELETE）：读取最新版本并加锁

class TokenBucket {
    double tokens_;          // 当前令牌数
    double rate_;            // 每秒产生令牌数
    double maxTokens_;       // 桶的最大容量
    std::chrono::steady_clock::time_point lastTime_;
    std::mutex mtx_;

public:
    TokenBucket(double rate, double max) 
        : tokens_(max), rate_(rate), maxTokens_(max),
          lastTime_(std::chrono::steady_clock::now()) {}

    bool allow() {
        std::lock_guard<std::mutex> lock(mtx_);
        auto now = std::chrono::steady_clock::now();
        double elapsed = std::chrono::duration<double>(now - lastTime_).count();
        tokens_ = std::min(maxTokens_, tokens_ + elapsed * rate_);
        lastTime_ = now;
        if (tokens_ >= 1.0) {
            tokens_ -= 1.0;
            return true;  // 允许通过
        }
        return false;  // 限流
    }
};

6.2 TCP 详解

流量控制（滑动窗口）

发送方根据接收方告知的**接收窗口大小（rwnd）**来限制发送量，防止接收方缓冲区溢出。

拥塞控制

四个阶段：

  1. 慢启动：从1个MSS开始，每收到一个ACK，cwnd（拥塞窗口）+1（指数增长）
  2. 拥塞避免：cwnd达到ssthresh后，每个RTT只增加1（线性增长）
  3. 快重传：收到3个重复ACK，立即重传丢失的段（不等超时）
  4. 快恢复：快重传后，ssthresh = cwnd/2，cwnd = ssthresh（从ssthresh线性增长，不回到1）

TCP 粘包与拆包

TCP是字节流协议，没有消息边界。发送"Hello"和"World"，接收方可能收到"HelloWorld"（粘包）或"He"+“lloWorld”（拆包）。

解决方案：

  1. 固定长度消息
  2. 消息头部包含长度字段（TLV协议）
  3. 分隔符（如HTTP的\r\n\r\n）
